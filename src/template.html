CONAMA's public home page - Desktop view

Motivation

Help improve the chorus organization by making the storage management easier delivering a product more accessible and feature-rich, while still learning new technologies and different work methodologies.

Objective

Replace the existing generic database with a solution that tackles and fixes the specific requirements and issues the old one had. The main two were: the ability to store files, in this case, digital versions of the music sheets, and being able to share an always updated list of the available pieces while keeping the edit permissions only to the CONAMA members.

The home page consists of a list of all pieces available, you can toggle which information is displayed at every moment, order it as you wish, or search for any parameter.

When you select one of them, a card with complete information will be displayed. Here, CONAMA members have the option to modify the piece or download the associated music sheets if it has one.

If you are logged in you can also access the section to create new pieces. The form to do so is designed to be flexible, meeting the required criteria in every case: you can remove versions, authors, arrangements, files, etc.

Intro

ESIMA (Estación de Sensado Inalámbrica para Motores de Aeromodelismo) is a wireless thrust stand prototype for brushless motors that started as a school project and is currently in mid-development. With my team, we are creating both the hardware and software from scratch, involving us in the whole production process, from UI/UX design to electronics, manufacturing, and testing.

Origin

When building hobby airplanes and drones there is an infinite combination of motors, propellers, speed controllers, batteries, etc. you can put together, so often is hard to predict how each arrangement will perform and the advantages and disadvantages of each setup.

This prototype is designed to streamline the process of testing and documenting the characteristics of each combination by measuring the thrust outputted by the propeller and the RPM while monitoring the current and tension of the used battery.

Web dev?

One of the parts I'm in charge of is of course the software. We are using the NodeMCU-32S microcontroller with the Arduino framework as the brain of the operation to read and control every sensor on board. The interesting part is that besides displaying information on the integrated OLED screen it can also connect to a WiFi network to host a web server locally. When you enter the given IP, you're prompted with a dashboard containing real-time graphs and a set of controls and from where you can command each and everything. Pretty neat!

As I said, this project is still under development, I'll shortly be posting updates.
